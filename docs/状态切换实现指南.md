# 状态切换与数据显示 — 实现指南

## 核心思路：叠加渲染，不销毁

暂停时**不销毁 PlayingState**，GameEngine 把旧状态存起来，暂停状态叠加渲染。

---

## 一、GameEngine 的状态管理

### 存储方式：状态栈

GameEngine 用一个栈来管理状态，而不是单个指针：

```
正常游戏时：
  栈底: [PlayingState]  ← 当前活跃

暂停时：
  栈底: [PlayingState]  ← 休眠，不 update，但要 render
  栈顶: [PausedState]   ← 当前活跃，处理输入

回到游戏：
  栈底: [PlayingState]  ← 恢复活跃
  （PausedState 被移除）
```

### GameEngine.h 改动

```cpp
#pragma once
#include <SFML/Graphics.hpp>
#include <vector>        // 用 vector 模拟栈
#include "InputHandler.h"

class GameState;

class GameEngine {
public:
    GameEngine();
    ~GameEngine();
    void run();

    // 状态管理
    void pushState(GameState* state);   // 压入新状态（旧状态休眠）
    void popState();                     // 弹出当前状态（恢复上一个）
    void changeState(GameState* state); // 替换当前状态（销毁旧的）

private:
    void processEvents();
    void update(float deltaTime);
    void render();

    sf::RenderWindow m_window;
    sf::Clock m_clock;
    InputHandler m_inputHandler;
    std::vector<GameState*> m_states;   // 状态栈
};
```

### GameEngine.cpp 实现

```cpp
#include "GameEngine.h"
#include "../states/GameState.h"

GameEngine::GameEngine()
    : m_window(sf::VideoMode(800, 800), "Pacman Game",
               sf::Style::Resize | sf::Style::Close) {
    m_window.setFramerateLimit(60);
}

GameEngine::~GameEngine() {
    // 释放所有状态
    for (auto* state : m_states) {
        delete state;
    }
}

void GameEngine::run() {
    while (m_window.isOpen()) {
        processEvents();
        update(m_clock.restart().asSeconds());
        render();
    }
}

// === 状态管理三个方法 ===

void GameEngine::pushState(GameState* state) {
    m_states.push_back(state);  // 压栈，旧状态不删除
}

void GameEngine::popState() {
    if (!m_states.empty()) {
        delete m_states.back();     // 删掉栈顶
        m_states.pop_back();        // 移除指针
    }
}

void GameEngine::changeState(GameState* state) {
    if (!m_states.empty()) {
        delete m_states.back();     // 删掉当前
        m_states.pop_back();
    }
    m_states.push_back(state);      // 放入新的
}

// === 主循环三步 ===

void GameEngine::processEvents() {
    sf::Event event;
    while (m_window.pollEvent(event)) {
        if (event.type == sf::Event::Closed ||
            (event.type == sf::Event::KeyPressed &&
             event.key.code == sf::Keyboard::Escape)) {
            m_window.close();
        }
        m_inputHandler.handleEvents(event);
        // 只有栈顶状态接收输入
        if (!m_states.empty()) {
            m_states.back()->handleInput(*this, event);
        }
    }
}

void GameEngine::update(float deltaTime) {
    // 只有栈顶状态执行 update
    if (!m_states.empty()) {
        m_states.back()->update(*this, deltaTime);
    }
}

void GameEngine::render() {
    m_window.clear(sf::Color(144, 238, 144));

    // 关键：从栈底到栈顶依次 render，实现叠加效果
    for (auto* state : m_states) {
        state->render(m_window);
    }

    m_window.display();
}
```

## 二、三种状态操作的区别

### changeState — 替换（菜单 → 游戏，游戏结束 → 重开）

```
之前: [MenuState]
之后: [PlayingState]    ← MenuState 被销毁
```

```cpp
// MenuState 里按 Enter 开始游戏
engine.changeState(new PlayingState());
```

### pushState — 叠加（游戏 → 暂停）

```
之前: [PlayingState]
之后: [PlayingState, PausedState]  ← PlayingState 休眠但保留
```

```cpp
// PlayingState 里按 Escape 暂停
engine.pushState(new PausedState());
```

### popState — 弹出（暂停 → 回到游戏）

```
之前: [PlayingState, PausedState]
之后: [PlayingState]    ← PausedState 被销毁，PlayingState 恢复
```

```cpp
// PausedState 里按 Escape 继续
engine.popState();
```

## 三、各状态的输入处理

### PlayingState::handleInput

```cpp
void PlayingState::handleInput(GameEngine& engine, sf::Event& event) {
    if (event.type == sf::Event::KeyPressed) {
        if (event.key.code == sf::Keyboard::Escape) {
            // 暂停：压入 PausedState，自己休眠
            engine.pushState(new PausedState());
        }
    }
}
```

### PausedState::handleInput

```cpp
void PausedState::handleInput(GameEngine& engine, sf::Event& event) {
    if (event.type == sf::Event::KeyPressed) {
        if (event.key.code == sf::Keyboard::Escape) {
            // 继续：弹出自己，PlayingState 恢复
            engine.popState();
        }
        if (event.key.code == sf::Keyboard::Q) {
            // 回主菜单：替换整个栈
            engine.changeState(new MenuState());
        }
    }
}
```

### MenuState::handleInput

```cpp
void MenuState::handleInput(GameEngine& engine, sf::Event& event) {
    if (event.type == sf::Event::KeyPressed) {
        if (event.key.code == sf::Keyboard::Enter) {
            // 开始游戏：替换为 PlayingState
            engine.changeState(new PlayingState());
        }
    }
}
```

### GameOverState::handleInput

```cpp
void GameOverState::handleInput(GameEngine& engine, sf::Event& event) {
    if (event.type == sf::Event::KeyPressed) {
        if (event.key.code == sf::Keyboard::Enter) {
            // 重新开始：替换为新的 PlayingState
            engine.changeState(new PlayingState());
        }
        if (event.key.code == sf::Keyboard::Q) {
            // 回主菜单
            engine.changeState(new MenuState());
        }
    }
}
```

## 五、暂停时的渲染效果

暂停时栈里有两个状态，render 从栈底到栈顶依次执行：

```
栈: [PlayingState, PausedState]

render 执行顺序：
1. PlayingState::render()   → 画地图、豆子、幽灵、Pacman（静止画面）
2. PausedState::render()    → 在上面叠加半透明遮罩 + "PAUSED" 文字
```

### PausedState::render 实现

```cpp
void PausedState::render(sf::RenderWindow& window) {
    // 半透明黑色遮罩覆盖整个窗口
    sf::RectangleShape overlay(sf::Vector2f(800, 800));
    overlay.setFillColor(sf::Color(0, 0, 0, 150));  // 半透明黑
    window.draw(overlay);

    // "PAUSED" 文字
    sf::Font font;
    font.loadFromFile("assets/fonts/arial.ttf");

    sf::Text text("PAUSED", font, 48);
    text.setFillColor(sf::Color::White);
    text.setPosition(300, 350);
    window.draw(text);

    sf::Text hint("ESC to resume, Q to quit", font, 24);
    hint.setFillColor(sf::Color::White);
    hint.setPosition(250, 420);
    window.draw(hint);
}
```

**效果：** 游戏画面变暗，中间显示 "PAUSED"，看起来像暂停了，但底层画面还在。

## 六、数据显示（UIManager）

UIManager 由 PlayingState 持有，每帧从数据源读取值，传给 render 显示。

### 数据流

```
ScoreManager.getScore() ──→ 1500
                              │
Pacman.getLives() ───────→ 2  │
                              │
PlayingState::render() ───────┤
                              │
    m_uiManager->render(window, 1500, 3000, 2)
                              │
                              ▼
                    屏幕: Score: 1500  Lives: 2
```

### UIManager.h

```cpp
#pragma once
#include <SFML/Graphics.hpp>
#include <string>

class UIManager {
public:

private:

    sf::Text m_scoreText;
    sf::Text m_highScoreText;
    sf::Text m_livesText;
};
```

### UIManager.cpp

```cpp
#include "UIManager.h"

UIManager::UIManager() {

                       int score, int highScore, int lives) {
    m_scoreText.setString("Score: " + std::to_string(score));
    m_highScoreText.setString("High Score: " + std::to_string(highScore));
    m_livesText.setString("Lives: " + std::to_string(lives));

    window.draw(m_scoreText);
    window.draw(m_highScoreText);
    window.draw(m_livesText);
}
```

### PlayingState::render 中调用

## 七、完整帧流程（暂停状态）

```
GameEngine::run() 主循环每一帧：

1. processEvents()
   │
   ├── 窗口事件（关闭/ESC）
   ├── m_inputHandler.handleEvents(event)
   └── m_states.back()->handleInput(*this, event)
       │
       PausedState::handleInput()
       ├── ESC → engine.popState()  弹出自己
       └── Q   → engine.changeState(new MenuState())

2. update(dt)
   │
   └── m_states.back()->update(*this, dt)
       │
       PausedState::update()
       └── 什么都不做（游戏暂停）

3. render()
   │
   ├── PlayingState::render()    ← 栈底，画游戏画面（静止）
   └── PausedState::render()     ← 栈顶，叠加半透明 + 文字
   │
   └── window.display()
```

## 八、编写顺序建议

1. **先改 GameEngine** — 实现 pushState/popState/changeState + m_states 栈
2. **写 MenuState** — 最简单，只处理 Enter 切换
3. **写 PlayingState 壳子** — 先能切换进去，内容后面填
4. **写 PausedState** — 叠加渲染 + ESC 切换
5. **写 GameOverState** — 同 MenuState 类似
6. **写 UIManager** — 接入 PlayingState 显示分数生命
